# iFare 基金會網站 — 維護 SOP

> 版本：v1.0（草案）  
> 建立日期：2026-04-14  
> 負責人：昀臻  

---

## 目錄

### 一、日常維護
- 1.1 每日檢查項目
  - 前台網站是否正常（i-fare.org.tw）
  - 後台是否可登入
  - API 是否回應正常
- 1.2 每週檢查項目
  - 訪客統計確認（Dashboard）
  - 錯誤 Log 檢查
- 1.3 每月檢查項目
  - 套件安全性更新（npm audit）
  - 資料庫備份確認
  - SSL 憑證到期日檢查

### 二、內容維護（後台操作）
- 2.1 新增/編輯最新消息
- 2.2 新增/編輯福利文章與懶人包
- 2.3 新增/編輯福利政策
- 2.4 代碼維護（新增類別/關鍵字等）
- 2.5 帳號管理（新增/停用管理員）
- 2.6 圖片管理

### 三、異常處理 SOP
- 3.1 前台網站無法存取
  - 排查流程：DNS → IIS → Node process → API → DB
- 3.2 後台無法登入
  - 排查流程：API 連線 → Token 過期 → 帳號狀態 → DB
- 3.3 API 回應錯誤
  - 排查流程：IIS 狀態 → Kestrel log → DB 連線 → 程式錯誤
- 3.4 資料庫連線失敗
  - 排查流程：SQL Server 服務 → 連線字串 → 防火牆 → 磁碟空間
- 3.5 頁面顯示異常（空白/排版亂）
  - 排查流程：Console 錯誤 → API 回應 → CSS 衝突 → 快取

### 四、定期維護作業
- 4.1 資料庫備份
  - 備份方式（SSMS / SQL Agent Job）
  - 備份頻率與保留策略
  - 備份檔存放位置
- 4.2 SSL 憑證更新
  - 憑證到期日
  - 更新流程
  - 負責人
- 4.3 網域管理
  - i-fare.org.tw 網域到期日
  - DNS 設定位置
  - 負責人
- 4.4 套件更新
  - npm audit 安全性檢查
  - 更新前的測試流程
  - 更新後的驗證 checklist

### 五、緊急應變
- 5.1 網站全站掛掉的緊急處理流程
- 5.2 資料庫損壞的復原流程
- 5.3 被入侵/資料外洩的應變流程
- 5.4 緊急聯絡人清單

### 六、交接 Checklist
- 6.1 帳號權限移交
- 6.2 文件確認清單
- 6.3 知識轉移會議建議議程
- 6.4 交接後觀察期注意事項
